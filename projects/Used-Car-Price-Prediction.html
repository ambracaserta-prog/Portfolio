The Challenge

Predict the sale price of used cars based on their features and conditions using real-world datasets. Daily Kaggle submissions were made over one month, with success measured by minimizing Root Mean Squared Error (RMSE) on unseen scoring data.

The core challenge was not just building accurate models — it was understanding why some models that performed well in training failed spectacularly on the leaderboard.

The Approach

01

Data Pre-processing

Cleaned datasets, addressed missing values via mean/median imputation grouped by car make, and extracted numeric values from character variables (e.g. torque lb). Generated correlation plots using ggcorrplot.

02

Feature Engineering

Transformed categorical variables into dummy variables. Engineered domain-knowledge features around fuel efficiency, body type, engine specs, and seller reputation.

03

Model Building

Explored a range from simple linear regression to ensemble methods using olsrr, caret, ranger, xgboost, and leaps. Applied tuning and feature pruning iteratively.

04

Iteration & Refinement

19 models submitted. Each iteration informed the next — adjusting mtry, tree count, node size, splitting rules, and predictor sets based on leaderboard feedback.

Contrasting Performance

Worst Performer

Model 3 — Random Forest (caret)

5,036 Test RMSE
24,810 Kaggle Score

Looked great in training — but a ~17,000 point gap on the leaderboard revealed severe overfitting and poor generalization to unseen scoring data.

Best Performer

Model 18 — Ranger Forest

3,733 Test RMSE
3,420 Kaggle Score

27 predictors, mtry=12, 1,000 trees, variance splitting. Adding torque_lb and make_name pushed generalization significantly — test and Kaggle scores finally aligned.

Key Learnings

🗂️

Structured Methodology

Properly defined objectives and preprocessing steps gave direction and focus throughout the month-long competition.

🔍

Data Exploration

Closely examining patterns revealed biases, outliers, and relevant features that raw statistics alone would have missed.

⚠️

Overfitting is Deceptive

A low training RMSE means nothing if the model can't generalize. Model 3 taught this lesson better than any textbook.

🔄

Iteration is Everything

Each of the 19 models built on the last. Small, deliberate changes — not big overhauls — drove the biggest improvements.

🧠

Domain Knowledge Matters

Understanding cars — what drives value, what buyers care about — shaped better features than algorithmic selection alone.

📊

Ensemble Methods Win

Random Forest and XGBoost consistently outperformed linear models by capturing non-linear relationships in the data.
