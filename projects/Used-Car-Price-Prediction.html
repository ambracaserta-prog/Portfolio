The Problem

The challenge was to accurately predict the market sale price of used cars based on a complex, multi-feature dataset. The primary business risk involved model overfitting and poor generalization — initial models performed well during training but failed significantly when evaluated on unseen scoring data.

The clearest sign of failure? A discrepancy of approximately 17,000 points between training RMSE and the Kaggle leaderboard score — a model that looked nearly perfect internally but collapsed on real-world data.

⚠️
The Core Risk
Model 3 achieved a training RMSE of 5,836 — but scored 24,810 on the Kaggle leaderboard. A 19,000-point gap revealed the model was capturing noise, not signal.

The Approach

I adopted a structured, iterative methodology to improve predictive accuracy and model robustness across four phases.

01

Preprocessing

Cleaned raw data using mean/median imputation grouped by car make. Extracted numeric values from character variables — e.g. converting raw torque strings into numeric torque_lb values.

02

Exploratory Data Analysis

Built a correlation matrix via ggcorrplot to visualize relationships between vehicle features — fuel economy, dimensions, horsepower — and sale price. Identified key signal variables early.

03

Modeling Techniques

Explored a full spectrum — from simple OLS linear regression to ensemble methods using ranger, caret, and xgboost. Applied 10-fold cross-validation to prevent bias throughout.

04

Hyperparameter Tuning

Systematically optimized mtry, min.node.size, and ntree. Switched from standard Random Forest to ranger for faster, more precise tuning across 19 iterations.

Key Insight & Outcome

This project underscored that predictive power is not merely a function of model complexity — it is a function of robust generalization.

While Model 3 achieved a low training RMSE of 5,836, its Kaggle score of 24,810 revealed a catastrophic failure to generalize. Through iterative refinement, Model 18 emerged as the highest-performing configuration — a tuned ranger forest with 27 predictors, mtry=12, and 1,000 trees — reducing leaderboard error to 3,420.

Engineering Maturity Lesson

This project taught me that modeling is an iterative learning process — not a linear task. Each of the 19 submissions built deliberately on the last.

📉

Validation Over Training Performance

Low training error is deceptive if the model is capturing noise rather than signal. Model 3 taught this lesson better than any textbook.

⚖️

The Simplicity–Performance Tradeoff

Complex models capture intricate patterns — but are prone to overfitting if not rigorously pruned and validated against unseen data.

🎯

Generalization Strategies

A "good" model maintains performance across unseen datasets. Feature selection and regularization matter more than raw model complexity.

🧠

Domain Knowledge Matters

Engineering torque_lb and consolidating make_name outperformed algorithmic tuning alone. Understanding cars drove better features than scaling models.

🔄

Iteration is Everything

Small, deliberate changes — not big overhauls — drove the biggest improvements. Each failure pointed directly to the next fix.

💬

Communicating Results

Presenting a 19,000-point gap clearly — not hiding it — is what separates a junior analyst from a senior one. Honest diagnostics build credibility.
